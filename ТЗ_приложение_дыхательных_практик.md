# Техническое задание: приложение для дыхательных практик (рабочее название — «Дыши»)

## 1. Цель продукта

Максимально лёгкое кроссплатформенное мобильное приложение (iOS + Android), которое ведёт пользователя через дыхательные техники с закрытыми глазами: голосом, звуком, метрономом и вибрацией. Пользователь выбирает технику, настраивает длительности фаз (там, где это допустимо), запускает сессию по количеству циклов или по таймеру. Есть лёгкая социальная механика: друзья видят факт практики друг друга и получают уведомления.

Ключевой принцип: **приложение должно работать с выключенным экраном и в фоне** — вся обратная связь дублируется аудио и вибрацией.

---

## 2. Каталог дыхательных техник

Каждая техника в приложении имеет: название, иконку/анимацию, краткое описание (2–4 предложения), пользу, паттерн фаз, диапазон настройки (если разрешён), противопоказания и технику безопасности.

### 2.1. Техники со счётом (структура «вдох — задержка — выдох — задержка»)

| Техника | Паттерн (сек) | Настройка длительностей | Примечания |
|---|---|---|---|
| Квадратное дыхание (Box) | 4-4-4-4 | Да, каждая фаза 2–10 с (обычно все равны) | Классика для снятия стресса; визуализация — квадрат |
| Треугольное дыхание | 4-4-4 (вдох-задержка-выдох) | Да, 2–10 с | Вариант без второй задержки; визуализация — треугольник |
| 4-7-8 (метод Э. Вейла) | вдох 4 — задержка 7 — выдох 8 | Пропорция фиксирована 4:7:8, но доступен множитель темпа ×0.5 / ×0.75 / ×1 / ×1.25 | Для засыпания и снижения тревоги. Новичкам — не более 4 циклов |
| 4-2-4 | вдох 4 — задержка 2 — выдох 4 | Да, все фазы 2–10 с | Мягкая балансирующая техника |
| 2-8 | вдох 2 — выдох 8 | Да, с сохранением или изменением соотношения 1:4 | Удлинённый выдох → активация парасимпатики, расслабление |
| 2-10 | вдох 2 — выдох 10 | Да, соотношение 1:5 | Более глубокая версия предыдущей |
| 4-16-8 | вдох 4 — задержка 16 — выдох 8 | Да, соотношение 1:4:2 масштабируется (например, 3-12-6, 5-20-10) | Пранаяма-соотношение; только для опытных. Упрощённый режим 4-8-8 предлагается по умолчанию |
| Когерентное дыхание | вдох 5–6 — выдох 5–6, без задержек | Да, 4–7 с на фазу | ~5–6 дыханий в минуту, выравнивание ВСР; добавить как «бонусную» технику |
| Метод Вима Хофа | Раунд: 30–40 глубоких дыханий → полный выдох → задержка на пустых лёгких (до позыва) → вдох + задержка 15 с. Обычно 3–4 раунда | Настраивается: число дыханий в раунде (20–50), темп дыханий, число раундов (1–5), длительность финальной задержки | Особая логика сессии (см. 2.3). Максимальные предупреждения безопасности |

### 2.2. Техники без счёта (свободный ритм + общий таймер)

| Техника | Таймер по умолчанию | Сопровождение |
|---|---|---|
| Диафрагмальное (брюшное) дыхание | 5 мин (настройка 1–30 мин) | Инструкция в начале (текст + голос), опциональный мягкий фоновый звук, гонг в конце |
| Попеременное дыхание ноздрями (Нади Шодхана) | 5 мин | Опциональные голосовые/звуковые подсказки смены ноздри: «левая… правая…» с настраиваемым интервалом, либо полностью свободный режим |
| Дыхание со звуком (Уджайи / Брамари «пчела») | 5 мин | Инструкция в начале, гонг в конце; для Брамари — подсказка «вдох / гудите на выдохе» опционально |

### 2.3. Особая логика для метода Вима Хофа

- Фаза 1: метроном/голос задаёт ритм активных дыханий, вслух или на экране идёт счёт (…28, 29, 30).
- Фаза 2: команда «выдохни и задержи» → секундомер задержки идёт вверх, пользователь прерывает задержку тапом в любой момент (или задаёт цель заранее).
- Фаза 3: «глубокий вдох, держи 15 секунд» → отсчёт → следующий раунд.
- После сессии показывается время задержек по раундам (личный прогресс).

### 2.4. Описание и техника безопасности (обязательный контент)

У каждой техники — экран «Описание», доступный до старта. Общие предупреждения (показываются при первом запуске приложения и в описании техник с задержками):

- Никогда не выполнять техники с задержкой дыхания и гипервентиляцией (особенно Вима Хофа) **в воде, за рулём, стоя, во время управления механизмами** — риск обморока.
- Головокружение, покалывание, туннельное зрение — сигнал остановиться и дышать нормально.
- Противопоказания для интенсивных техник и длительных задержек: беременность, эпилепсия, сердечно-сосудистые заболевания, гипертония, панические расстройства, недавние операции, глаукома. Перед практикой — консультация врача.
- Приложение не является медицинским изделием и не заменяет медицинскую помощь (дисклеймер в онбординге и настройках).
- Для 4-7-8 и 4-16-8: начинать с 3–4 циклов, увеличивать постепенно.

## 3. Настройка сессии

Перед стартом любой техники пользователь задаёт:

1. **Режим окончания** (одно из двух):
   - количество циклов (1–100), или
   - таймер (1–60 мин) — приложение само досчитывает целое число циклов и завершает по ближайшему концу цикла.
2. **Длительности фаз** — только для техник, где это разрешено (см. таблицу). Слайдеры с шагом 0.5 с, кнопка «Сбросить к классике».
3. **Тип сопровождения** (можно комбинировать):
   - **Голос**: озвучивание фаз словами «вдох», «задержка», «выдох», «задержка». Опционально — голосовой отсчёт секунд («вдох… 2… 3… 4»). Выбор голоса (мужской/женский), языки: русский + английский на старте.
   - **Звук**: разные короткие сигналы на каждую фазу (например, восходящий тон = вдох, ровный = задержка, нисходящий = выдох), чтобы практиковать с закрытыми глазами без слов. 3–5 звуковых наборов: «Дерево», «Чаша», «Капля», «Минимал (бип)», «Природа».
   - **Метроном**: тикающий каждую секунду метроном с выбором звука тика и акцентом на смене фазы. Отключаем.
   - **Вибрация**: паттерны вибрации на смену фазы (короткая = вдох, двойная = задержка, длинная = выдох) — режим для полной тишины.
   - **Визуальный режим**: анимированная фигура (расширяющийся/сжимающийся круг; для квадрата — движение точки по периметру квадрата, для треугольника — по треугольнику) + крупная надпись текущей фазы + отсчёт.
4. **Подготовка**: отсчёт 3–5 секунд перед первым вдохом («приготовьтесь…»).
5. Гонг/колокол в конце сессии.

Настройки сессии сохраняются per-техника (последние использованные).

## 4. Социальные функции

- **Аккаунт**: регистрация по e-mail, Google, Apple ID (обязателен для соцфункций; практиковать можно и без входа — гостевой режим, данные локально).
- **Друзья**: поиск по нику, добавление по заявке (двустороннее подтверждение), удаление, блокировка.
- **Лента друга**: видно только факт практики: техника, длительность, дата/время, текущая серия дней (streak). Никаких подробных данных здоровья.
- **Уведомления**: push «Аня завершила практику „Квадратное дыхание“, 10 минут 🌬️» — включается/отключается per-друг и глобально.
- **Приватность**: режимы «делиться с друзьями / не делиться»; шеринг можно отключить для отдельной сессии перед стартом.
- Опционально (фаза 2): взаимные «дай пять», совместные челленджи («7 дней подряд»).

## 5. Статистика и мотивация

- Календарь практик, суммарные минуты, число сессий, streak.
- Для Вима Хофа — график времени задержек.
- Локальное напоминание «пора подышать» (настраиваемое время).

---

## 6. Экраны (MVP)

1. Онбординг (3 экрана: что это, безопасность/дисклеймер, вход или гостевой режим).
2. Главный: сетка/список техник с иконками, поиск, «Недавние».
3. Карточка техники: описание, польза, безопасность, кнопка «Начать».
4. Настройка сессии (см. раздел 3).
5. Экран сессии: анимация, фаза, отсчёт, номер цикла / оставшееся время, пауза/стоп. Экран не гаснет (wakelock), но всё дублируется звуком.
6. Итог сессии: длительность, циклы, streak, кнопка «поделиться с друзьями».
7. Друзья: список, заявки, лента активности.
8. Профиль и настройки: голос, звуки, вибрация, уведомления, приватность, язык, «О приложении» + дисклеймер.

## 7. Нефункциональные требования

- Размер приложения: цель ≤ 30 МБ (аудио-ассеты в ogg/aac, минимум растровой графики — иконки и анимации в векторе: SVG/Rive/Lottie).
- Полная работа офлайн для практик; сеть нужна только для соцфункций.
- Работа в фоне и при заблокированном экране: аудио-сессия не прерывается, таймеры фаз точные (не setTimeout-подобные, а на аудио-движке/нативных таймерах). Допустимый дрейф — не более ±50 мс на фазу.
- Корректное поведение при звонке (пауза сессии), при подключённых наушниках/Bluetooth.
- Тёмная и светлая темы. Локализация: RU, EN.
- Точность важнее TTS-в-реальном-времени: голосовые команды («вдох», «выдох», цифры 1–20) **предзаписываются/предгенерируются в аудиофайлы** и проигрываются как сэмплы — это устраняет задержку запуска системного TTS. Системный TTS — только запасной вариант.

## 8. Технологический стек и библиотеки

### 8.1. Рекомендуемый стек: Flutter

| Задача | Пакет | Комментарий |
|---|---|---|
| Метроном | `metronome` | Точный кроссплатформенный метроном: BPM, громкость, свои аудиофайлы тиков, tick-callback для синхронизации UI |
| Проигрывание звуков/голоса | `just_audio` (+ `audio_session`) | Гэплесс-плейлисты, любые источники; audio_session — корректное поведение со звонками и другими приложениями |
| Низколатентные короткие сэмплы | `flutter_soloud` или `audioplayers` (low latency mode) | Для сигналов смены фаз, где важны миллисекунды |
| Голос (запасной вариант) | `flutter_tts` | Нативные движки iOS/Android/Web, регулировка скорости/тона; умеет `synthesizeToFile` — можно предгенерировать фразы прямо на устройстве |
| Фоновое аудио | `just_audio_background` / `audio_service` | Управление с локскрина, продолжение сессии в фоне |
| Вибрация | `vibration` / `haptic_feedback` | Кастомные паттерны вибрации |
| Не гасить экран | `wakelock_plus` | На экране сессии |
| Анимации | `Rive` (`rive`) или `Lottie` (`lottie`) + CustomPainter | Дыхательные круги/квадраты; Rive — интерактивные state machines, очень лёгкие файлы |
| Локальная БД | `isar` или `drift` / `hive` | История сессий офлайн |
| Push-уведомления | `firebase_messaging` + `flutter_local_notifications` | Уведомления о практике друзей + локальные напоминания |
| Аутентификация и бэкенд | Firebase (Auth, Firestore, Cloud Functions) **или** Supabase (Auth, Postgres, Realtime, Edge Functions) | Supabase дешевле и открытее; Firebase проще для push. Cloud Function/Edge Function рассылает push друзьям по событию «сессия завершена» |
| Аналитика (опц.) | Firebase Analytics / PostHog | |

### 8.2. Альтернативный стек: React Native (Expo)

`expo-audio` (звук), `react-native-tts` (голос), `notifee` + FCM (push), `react-native-reanimated` + `lottie-react-native` (анимации), Supabase/Firebase (бэкенд). Минус: точный тайминг аудио сложнее, чем во Flutter — потребуется нативный модуль для метронома.

### 8.3. Генерация и источники ассетов

**Звуки (тики метронома, сигналы фаз, гонги):**
- Библиотеки бесплатных звуков: Freesound.org (проверять лицензию CC0/CC-BY), Pixabay Sound Effects, Zapsplat.
- Синтез собственных сигналов: Audacity (генератор тонов), sfxr/jsfxr, Tone.js/SuperCollider — сгенерировать чистые восходящие/нисходящие тоны под фазы, экспорт в ogg 48 кбит/с.
- Тибетские чаши/гонги — записи CC0 либо генерация в DAW.

**Голосовые фразы («вдох», «задержка», «выдох», цифры, инструкции):**
- Предгенерация через нейро-TTS: ElevenLabs (лучшее качество, есть русский), Google Cloud TTS (WaveNet, ru-RU), Yandex SpeechKit (отличный русский), Azure TTS. Один раз сгенерировать полный набор фраз для каждого голоса → сложить в ассеты.
- Либо запись диктора (2 голоса × ~40 фраз × 2 языка).

**Графика и анимация:**
- Иконки техник: SVG (Figma), генерация иконок/иллюстраций — Recraft (умеет векторный вывод), Midjourney/DALL-E/Stable Diffusion для растровых фонов (использовать умеренно из-за размера приложения).
- Дыхательные анимации: Rive (rive.app, бесплатный редактор, файлы по несколько КБ) или готовые Lottie с LottieFiles (проверять лицензию).

**Полезно при разработке с ИИ-ассистентами (Claude Code и т.п.):** скиллы/инструменты генерации фронтенда и дизайна (frontend-design), генерация аудио-ассетов скриптами (Python: `numpy` + `scipy.io.wavfile` для синтеза тонов), автоматическая генерация TTS-фраз через API одним скриптом.

## 9. Модель данных (минимум)

- **User**: id, nickname, email, avatar(опц.), settings, privacy_flags.
- **Friendship**: user_a, user_b, status (pending/accepted/blocked).
- **Technique** (справочник, зашит в приложение): id, name, type (counted/timer/wim_hof), default_pattern, editable_phases (bool per фаза), min/max per фаза, description, safety_text, safety_level.
- **Session**: id, user_id, technique_id, started_at, duration_sec, cycles, pattern_used (json), wim_hof_retentions (json, опц.), shared (bool).
- **NotificationSetting**: user_id, friend_id, enabled.

## 10. Этапы

**MVP (этап 1):** гостевой режим, 8 техник со счётом + 3 без счёта, циклы/таймер, голос + звук + метроном + вибрация + анимация, локальная статистика, напоминания, RU/EN.

**Этап 2:** аккаунты, друзья, лента, push «друг попрактиковал», Вим Хоф с графиком задержек, streak-челленджи.

**Этап 3:** виджеты, Apple Watch/Wear OS, интеграция Apple Health / Google Fit (mindful minutes), новые звуковые наборы.

## 11. Критерии приёмки (ключевые)

1. Сессия «квадрат 4-4-4-4, 10 циклов» проходит с точностью фаз ±50 мс, с выключенным экраном, только по звуку.
2. Голосовой режим на русском корректно произносит фазы без запаздывания (>200 мс от начала фазы — баг).
3. При входящем звонке сессия ставится на паузу и корректно возобновляется.
4. Push другу приходит в течение 60 секунд после завершения расшаренной сессии.
5. Размер релизной сборки Android ≤ 30 МБ.
6. Дисклеймер безопасности показан до первой практики и доступен из настроек; техники с задержками показывают предупреждение при первом запуске.
