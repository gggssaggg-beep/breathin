# System review 2026-07-16: вход, стрики, челленджи, UX

> Ревью Fable по запросу владельца («придумай как оптимизировать интерфейс,
> сделать удобнее; посмотри систему залогинивания, стриков, челленджей»).
> Формат: сначала находки-дефекты, затем ранжированные UX-предложения.
> Реализация — после утверждения владельцем (правило: ревью до фиксов).

## 1. Дефекты и технические долги

### Д1 (СУЩЕСТВЕННО). Задержки Вима Хофа не синкаются в облако
`session_sync_service.dart` не включает `retentionsSec` ни в select, ни в
upsert-payload — колонки нет и в `docs/supabase/schema.sql`. Следствие:
после переустановки/смены устройства история вернётся, а **график прогресса
ВХ будет пуст** (retentionsSec = null у всех записей из облака). Данные
копятся с v0.6.0 и теряются молча.

Фикс: колонка `retentions int[]` в schema.sql (миграция владельцем), payload
и select по образцу `variant` (тот же трюк с откатом при отсутствии колонки).

### Д2 (мелочь). У таймер-сессий и ВХ синк вызывает `syncNow`, а вот
`variant` у таймера null — это НОРМА (сабтайтл возьмёт минуты), но в облаке
таймер-запись неотличима от «старой» counted-записи без варианта. Не дефект,
зафиксировано как осознанное решение плана §1.7.

### Д3 (наблюдение). Стрик считается по локальной дате устройства — при
перелёте через часовые пояса серия может «сгореть» или задвоиться. Осознанный
компромисс (ТЗ §7: без сети), чинить не предлагаю — только зафиксировать.

## 2. UX: главный экран и путь до практики

### У1 (САМОЕ ЦЕННОЕ). «Быстрый старт» — повторить последнюю практику
Сейчас путь до дыхания: карточка → описание → (прокрутка) → кнопка →
setup → «Начать» = **4–5 тапов каждый раз**. Для ежедневного пользователя
описание техники и настройки не меняются неделями.

Предложение: под стрик-баннером (или вместо пустого места до серии) —
карточка «Продолжить: <Техника> · <паттерн> · <N мин>» с кнопкой ▶,
запускающей сессию с последними настройками В ОДИН ТАП (мимо карточки и
setup). Данные уже есть: последняя запись лога + сторы настроек per-техника.

### У2. Группировка каталога: 19 техник — одна длинная сетка
Прокрутка до нижних техник длинная, различие «успокаивающее/бодрящее/
суфийское» видно только по иконке. Предложение — лёгкие секции-заголовки в
сетке (Избранное / Спокойствие / Энергия / Суфийские / Особые) ИЛИ чипы-
фильтры над сеткой. Избранное уже поднимает наверх — секции дополняют, не
ломают.

### У3. Пустое состояние стрика
До первой практики главный экран не мотивирует. Дешёвый вариант: при
streak == 0 и непустой истории показывать «Серия прервалась — начните
заново сегодня» (то же место, приглушённый цвет). Для полного новичка — не
показывать ничего (не давить).

## 3. Вход и сохранность данных

### В1. Пользователь не знает, что история локальная
Синк работает ТОЛЬКО у вошедших; гость теряет всё при переустановке — и
узнаёт об этом постфактум. Предложение: в «Практике» после ≥10 сессий у
гостя — ненавязчивая карточка «История хранится только на этом устройстве.
Войдите — и она сохранится» с кнопкой в настройки. Один раз, закрываемая.

### В2. Порядок кнопок входа
Сейчас первым идёт «Создать профиль (без регистрации)» — гостевой аккаунт,
который при выходе невосстановим. Для долгосрочной сохранности выгоднее
подталкивать к e-mail/Google. Предложение: первым — e-mail (главный
постоянный способ), гостевой — вторым с подписью «можно привязать почту
позже» (linkEmailIdentity уже реализован).

## 4. Стрики (механика)

### С1. Напоминание «серия сгорит»
Самая работающая duolingo-механика. Вечером (например, 20:00 локального),
если сегодня практики не было и streak > 0 — локальное уведомление «Огонёк
ждёт: 5 дней подряд. Подышите 3 минуты». Инфраструктура разрешений
(notification_permission.dart) уже есть; нужен flutter_local_notifications
+ ежедневный пересчёт при запуске. Средний объём работы.

### С2. Заморозка серии (streak freeze)
1 «заморозка», зарабатываемая, например, 7 днями подряд: пропущенный день
списывает её, а не серию. Против: усложняет честную механику. Предлагаю НЕ
делать, пока стрики не приживутся (решение владельца).

## 5. Челленджи

### Ч1. Прогресс обновляется только при заходе на экран
`loadAndSyncProgress` дергается при открытии «Челленджей». Если пользователь
не заходит туда — соперники видят его нулевой прогресс. Дешёвый фикс:
дергать синк прогресса челленджей после каждой записи сессии (рядом с
`SessionSyncService().syncNow()`), fire-and-forget.

### Ч2. Идентификация участников
Ники есть (display_name), но гость по умолчанию «Гость» — в таблице из трёх
«Гостей» непонятно кто есть кто. Дешёвый фикс: при создании/входе в челлендж
гостю с дефолтным ником предлагать сразу задать имя (диалог уже есть в
настройках — переиспользовать).

## 6. Мелкий UX-полиш (дёшево, по одному коммиту)

* **П1.** На финише counted-сессии нет итога «N минут · M циклов» — только
  галочка. Одна строка добавит ощущения результата (у ВХ уже есть задержки).
* **П2.** BOLT-карточка в «Практике» — самый верх, хотя открывают экран
  ради календаря. Поменять местами: календарь выше, BOLT ниже итогов месяца.
* **П3.** В настройках нет «Оценить/поделиться приложением» — для органики
  пригодится, когда появится страница в сторе (отложить).
* **П4.** Звук подсказки ноздри в таймере: добавить подпись в setup
  «ниже тоном — левая, выше — правая» (иначе семантику нот надо угадывать).
