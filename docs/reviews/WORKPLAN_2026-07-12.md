## Партия 2 — `fix/audio-memory` (К2, К3, М1, М2, С8-сессия)

1. **Чанковый рендер** (`lib/services/audio/timeline_renderer.dart`): сейчас
   `render()` аллоцирует `Int32List(totalSamples)` + `Int16List` — сотни МБ на длинных
   сессиях → OOM (доступно из UI: таймер 60 мин). Добавить
   `void renderInto(SessionPlan plan, SoundBank bank, Int16List chunk, int chunkStartSample)`
   ИЛИ метод `Iterable<Int16List> renderChunks(plan, bank, {int chunkSamples})`:
   события отсортированы — для каждого чанка микшируй только клипы, пересекающие окно
   (два указателя; аккумулятор Int32List размера чанка, клamp в int16). Существующий
   `render()` перепиши через чанки (конкатенация) — старые тесты должны пройти без правок.
2. **Потоковая запись WAV**: в `wav_io.dart` добавить
   `WavStreamWriter(IOSink, sampleRate)` нельзя (dart:io в чистом модуле нежелателен) —
   лучше: в `session_audio_builder.dart` новая функция
   `Future<void> writeSessionWavFile(SessionPlan, SoundBank, FeedbackChannels, File out)`:
   пишет 44-байтовый заголовок (длина известна заранее: totalSamples как в renderer,
   вынести расчёт длины в публичный метод рендерера) + чанки по ~10 с через
   `RandomAccessFile`/`IOSink.add`. `buildSessionWav` (in-memory) оставить для тестов
   и коротких сессий, но SessionRunner перевести на файловый путь.
3. **Файл сессии** (`session_runner.dart`): имя фиксированное `session_current.wav`
   (перезапись), удаление в `dispose()` fire-and-forget (`File(...).delete().ignore()`
   с try/catch) — кэш не растёт (К3).
4. **Вибро-фиксы** там же: `int _lastMs = -1;` (событие t=0 попадает в окно — М1);
   в `_onTick` если `pos - _lastMs > 2000` — пропустить вибрацию окна (возврат из фона,
   залп — М2), просто `_lastMs = pos; return` до диспетчеризации вибро (визуал обновить).
5. **Синк после сессии**: в `_record()` после `log.add(record)` — fire-and-forget
   `SessionSyncService().syncNow()` (импорт; ошибки внутри синка уже глотаются) (С8).
6. Тесты: на чанковый рендер (сравнить с эталонным render() на плане с перекрытиями
   клипов и на границах чанков), на writeSessionWavFile (tmp-файл → WavIo.decode →
   сэмплы == render()). Существующие 184+ теста зелёные.

## Партия 3 — `feat/session-pause-finish` (С1, С2, С3, М3, М8 + влад. §9, §14)

1. **wakelock_plus** в pubspec (`wakelock_plus: ^1.2.0`); в SessionRunner:
   `WakelockPlus.enable()` в initState (try/catch — тестов без плагина не ронять),
   `WakelockPlus.disable()` в dispose (ПЛАН §6.5: экран не гаснет в визуальном режиме).
2. **Пауза/резюм** (ПЛАН §3.3 п.5): состояние `_paused` в _SessionRunnerState.
   - Пауза: аудио-режим — `handler.pause()`; визуальный — `_clock.stop()`. Тикер живёт.
   - Резюм: позиция = начало ТЕКУЩЕЙ фазы: из `_engine.stateAt(pos)` взять
     `pos - state.phaseElapsedMs`; аудио — `await handler.seek(Duration(...)); handler.play()`;
     визуальный — Stopwatch не умеет seek: держать поле `int _baseMs` (смещение),
     позиция = `_baseMs + _clock.elapsedMilliseconds`; при резюме `_baseMs =
     начало фазы; _clock..reset()..start()`. После резюма `_lastMs = позиция`
     (не переигрывать вибро прошедших событий).
   - Пауза с локскрина/звонок: just_audio сам ставит паузу (handleInterruptions) —
     в `_onTick` если `_audioMode && !handler.player.playing && !_paused && !finished`
     → отразить паузу в UI (`_paused = true`).
   - Стоп с локскрина (М3): подписаться на `handler.playbackState`, при
     `processingState == idle` (после stop) — если не финиш, записать прерванную
     и `maybePop` (как _stop).
3. **Кнопки** (session_view.dart): вместо одной «Пауза / стоп» — две:
   пауза/продолжить (иконка playerPause/playerPlay — ДОБАВИТЬ в breathin_icons.dart
