## Часть 1. Таймерный режим движка (П10)

### 1.1. Зачем и что сейчас

Четыре техники каталога — `diaphragmatic` (диафрагмальное), `nadi_shodhana`
(попеременное), `sound_breath` (со звуком), `axis` (по оси) — имеют
`TechniqueType.timer` и **не запускаются**: `canStart` в карточке пускает
только counted/scripted/wimHof, кнопка показывает «Скоро». Это последние
«мёртвые» техники каталога (4 из 19).

Суть таймер-техники: свободное дыхание без счёта фаз, заданная длительность
(N минут), опциональные периодические подсказки (Нади Шодхана: смена ноздри
каждые 10–30 с), гонг в конце.

### 1.2. Ключевое архитектурное решение — БЕЗ пре-рендеренного WAV

Counted-сессии строят единый PCM-таймлайн (WAV на диск) — это даёт ±50 мс
и жизнь при погасшем экране. Для таймер-сессий этот путь НЕ годится:

* 30-минутная сессия = ~158 МБ несжатого WAV на диск при почти сплошной
  тишине — расточительно и медленно;
* точность подсказок «смени ноздрю» — секунды, а не миллисекунды: строгий
  таймлайн не нужен.

**Решение:** аудио-якорем сессии становится **фоновый трек-луп**
(`background_loop.ogg`, уже в ассетах) через наш `audio_service`-handler —
он держит foreground-сервис (жизнь при погасшем экране, пауза/стоп с
локскрина), а подсказки и гонг играются **one-shot'ами** по Dart-часам
(Stopwatch), как уже сделано в Вим Хофе. Погрешность one-shot (десятки мс)
для подсказок несущественна.

Риск: doze/battery-оптимизации Android могут душить Dart-таймеры при
погасшем экране. Митигация: активный foreground-плеер (наш случай) держит
процесс исполняемым — тот же механизм, что у counted. Проверить на
устройстве владельца ПЕРВОЙ ручной проверкой. План Б, если тики душатся:
позиция фонового плеера (`player.position` по лупу с учётом числа циклов)
как часы вместо Stopwatch. План В (крайний): пре-рендер WAV чанками на диск
допустим, но только как последний выход.

### 1.3. Доработка audio_service-handler

Сейчас handler умеет `loadSessionFile(файл)`. Добавить:

* `loadLoopingAsset(String asset, {String title, Duration? sessionDuration})`
  — asset-источник с `LoopMode.all`; медиа-уведомление как у сессий;
* пауза/резюм/стоп с локскрина работают как есть (те же коллбеки);
* у таймер-сессии `duration` уведомления = длительность сессии, прогресс —
  по Dart-часам (не по позиции лупа) — проверить, что наш handler позволяет
  задать это вручную; если нет — без прогресса в уведомлении (не критично).

Фоновый плеер `_bgPlayer` из counted-сессий тут НЕ используется — фон и есть
основной источник (не плодим два плеера на один и тот же луп).

### 1.4. Домен

Новый файл `lib/domain/engine/timer_session.dart` (чистый Dart, юниты):

```dart
class TimerSessionConfig {
  final int minutes;            // из setup, клэмп min/maxTimerMin техники
  final int prepSeconds;        // 0..5, как у counted
  final int cueIntervalSec;     // 0 = подсказки выключены (только нади)
}

enum TimerCue { left, right }   // Нади Шодхана: с ЛЕВОЙ начинают классически

class TimerCueEvent { final int tMs; final TimerCue cue; }

/// Чистая функция: расписание подсказок на сессию.
/// Первая подсказка — на конце prep (старт практики), далее каждые
/// cueIntervalSec, чередуя left/right начиная с left. Последняя подсказка
/// не ставится, если до гонга остаётся < cueIntervalSec/2 (обрубок).
List<TimerCueEvent> cueSchedule(TimerSessionConfig c);
```

Состояние сессии — НЕ PhaseEngine (фаз нет): маленькая машина
`TimerSessionMachine` по образцу `WimHofMachine` (`advance(dtMs)`,
`stage: prep|breathing|finished`, `elapsedMs`, `currentCue`,
`nextCueInMs`) — тестируется юнитами с переносом остатка dt.

### 1.5. Экран сессии (`lib/features/timer_session/`)

`TimerSessionScreen` — свой, по образцу `WimHofSessionScreen` (тикер +
машина). ВАЖНО UX: свободное дыхание — фигура НЕ должна навязывать ритм:

* prep: «Приготовьтесь» + отсчёт (как везде);
* practice: спокойная фигура техники (круг) с **медленным мягким свечением**
  (период ~8 с, амплитуда малая — «живая», но не дирижирует дыханием);
  крупно — прошедшее/оставшееся время; для Нади Шодханы — текущая метка
  «Левая ноздря» / «Правая ноздря» крупным текстом + смена по подсказке;
* пауза/стоп — как в counted (кнопки те же);
* финиш: гонг one-shot, галочка, тап — на каталог (popUntil first);
* wakelock: как в counted.

Каналы сопровождения:

* **звук** = фоновый луп (якорь) + one-shot подсказок + гонг.
  Звук подсказки ноздри: две ноты живой арфы из лесенки — `note_1` (D4,
  ниже = ЛЕВАЯ) и `note_5` (C5, выше = ПРАВАЯ) — консистентно со звуком
  приложения, без новых ассетов. Не-нади техники подсказок не имеют;
* **вибро** = на подсказке: левая — 1 короткий импульс, правая — 2
  (различимо с закрытыми глазами); на гонге — длинный;
* **метроном** для таймер-техник смысла не имеет — тумблер скрыть;
* при выключенном звуке — чистый визуал-режим (Stopwatch, как counted).

### 1.6. Setup-экран (`TimerSetupScreen`)

Свой (SessionSetupScreen заточен под фазы):

* слайдер «Длительность» minTimerMin..maxTimerMin (дефолт из техники);
* для nadi_shodhana — чипы интервала подсказок из `periodicCue`
  (Выкл/10/15/20/30 с, дефолт 15);
* подготовка (3..5 с) и каналы (звук/вибро) — как у counted;
* персист — `TimerSettingsStore` (prefs JSON per-техника, клэмп диапазонов
  при чтении — по образцу WimHofSettingsStore);
* маршрут: `technique_card_screen._startSession` → тип timer →
  TimerSetupScreen; `canStart` разрешает timer при `defaultTimerMin != null`.

### 1.7. История и статистика

`SessionRecord`: techniqueId, durationSec = фактическое, cyclesCompleted = 0,
completed по гонгу, `variant: '15 мин'`? — НЕТ: variant хранит паттерн фаз,
для таймера оставить null (сабтайтл статистики возьмёт минуты из
durationSec). Прерывание раньше 1 минуты — не записывать (мусор).
Существующая статистика (календарь/минуты) подхватит записи автоматически.

### 1.8. Тесты (минимум)

* `cueSchedule`: интервалы, чередование с левой, «обрубок» не ставится,
  cueIntervalSec=0 → пусто;
* `TimerSessionMachine`: границы prep/practice/finish, перенос dt, пауза;
* `TimerSettingsStore`: roundtrip/клэмп/битый JSON;
* виджет-поток: prep → практика (метка ноздри сменяется по времени) →
  финиш → запись в лог; стоп до минуты — записи нет;
* карточка: 4 таймер-техники стартуют (canStart), «Скоро» не показывается.

### 1.9. Партии и делегирование

1. **Партия T1 (Fable):** домен (`timer_session.dart`) + handler
   `loadLoopingAsset` + юниты. Тонкая логика — не делегировать.
2. **Партия T2 (sonnet по спеке, ревью Fable):** TimerSetupScreen +
   TimerSettingsStore + маршрутизация карточки + l10n (RU/EN: «Длительность»,
   «Подсказки смены ноздри», «Левая/Правая ноздря» — segNadi ключи) + тесты.
3. **Партия T3 (Fable):** TimerSessionScreen (машина+тикер+аудио) + вибро +
   финиш/запись + виджет-тесты. Ручная проверка на телефоне: погасший экран
   15 минут, подсказки не отстают (риск 1.2).

## Часть 2. График задержек Вима Хофа (П19)

### 2.1. Данные — уже копятся

`SessionRecord.retentionsSec: List<int>?` пишется каждой ВХ-сессией
(с v0.6.0). Ничего мигрировать не надо; у старых записей поле null —
фильтруются.

### 2.2. Агрегация (чистый Dart)

`lib/domain/stats/wim_hof_stats.dart`:

```dart
class WimHofSessionStat {
  final DateTime date;
  final int bestSec;      // max по раундам сессии
  final List<int> rounds; // все раунды (тултип/детали)
}

/// Хронологический список ВХ-сессий с задержками (retentionsSec != null,
/// непустой). Плюс хелперы: bestEver(), последняя сессия.
List<WimHofSessionStat> wimHofProgress(List<SessionRecord> log);
```

Тесты: фильтрация чужих техник и null, сортировка по дате, bestSec.

### 2.3. Переиспользование графика

`BoltChart` (спарклайн CustomPainter) обобщить: вынести в
`lib/ui/charts/sparkline_chart.dart` — `SparklineChart(values: List<int>,
maxPoints)`; `BoltChart` становится тонкой обёрткой (bolt-тесты не трогаем,
поведение то же). ВХ-график — та же обёртка по `bestSec`.

### 2.4. Где показывать (два места, обе точки дешёвые)

1. **Карточка техники ВХ** (`technique_card_screen`, только для
   `wim_hof`): между «Пользой» и «Безопасностью» секция «Ваш прогресс»:
   спарклайн по bestSec последних 30 сессий + строка «Лучшая задержка: N с»
   + подпись последней сессии «раунды: 45 / 62 / 58 с». Пустое состояние —
   секция не показывается вовсе (не мозолит новичку).
2. **Финиш-экран ВХ** (`_FinishedView`): под итогом раундов — одна строка
   сравнения: «Лучшая за сегодня: 62 с · Ваш рекорд: 71 с» (или «Новый
   рекорд!» с огоньком, если побит). Без графика — только мотивирующая
   строка.

Загрузка: `SessionLogRepository.all()` уже используется в карточках — в
technique_card_screen добавить FutureBuilder только для wim_hof (не грузить
лог на остальных 18 карточках).

### 2.5. Тексты (RU/EN)

`whProgressTitle` «Ваш прогресс» / "Your progress";
`whBestLabel` «Лучшая задержка: {sec} с»;
`whLastRounds` «Последняя сессия: {list}»;
`whNewRecord` «Новый рекорд!» / "New record!";
`whRecordCompare` «Лучшая за сегодня: {today} с · Рекорд: {best} с».

### 2.6. Партии

* **Партия G1 (sonnet по спеке, ревью Fable):** wim_hof_stats + вынос
  SparklineChart + секция на карточке + тесты. Механика, хорошо
  специфицирована.
* **Партия G2 (Fable, мелкая):** строка рекорда на финише ВХ (трогает
  живой экран сессии — лучше руками) + тест.
